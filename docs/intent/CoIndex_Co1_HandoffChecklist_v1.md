# CoIndex – Co1 Handoff Checklist v1
Mission Id   : CoIndex_BackboneDesign2025_Mission
Session label: MurphyPlus_CoIndex20251123
Stamp        : 20251124T025143Z

## 1. Scope

This checklist defines the minimum conditions for Co1 to assume
long-term stewardship of CoIndex.

## 2. Design Surfaces

[ ] README and Vision documents are present and current  
[ ] C–I–X–S–K object model defined and linked  
[ ] CoPointee spec and Canonality spec present  
[ ] Governance and Reputation models present  
[ ] Night-job architecture and Evolution pipeline documented  
[ ] CoPortal + GIBindex overlay designs present  
[ ] CoCache migration plan and deprecation policy present  

## 3. Operational Surfaces

[ ] Docs/ops/MegaWave/ file reflects actual waves completed  
[ ] BPOE notes exist for countdown style and workflow  
[ ] Locations for nightjob receipts and migration runs are fixed  
[ ] CoPortal and GIBindex doc paths are stable  

## 4. Governance Readiness

[ ] Canonality levels accepted by Co1  
[ ] Reputation layers compatible with MeritRank / CoHalo / RepTag  
[ ] Edge approval rules documented and acceptable  
[ ] CoSteward seed-stage overrides clearly limited  

## 5. Integration Points

[ ] CoIndex linked from CoSteward operations manual (planned)  
[ ] Hooks to CoCore / CoPortal / GIBindex identified  
[ ] CoCache index migration staging plan approved  
[ ] Co1 aware of nightjob architecture and limits  

## 6. Safety and Anti-Capture

[ ] No single vendor/platform has exclusive control over semantics  
[ ] Deprecation policy for legacy indexes agreed  
[ ] Audit trails for canonality and reputation changes available  

## 7. Acceptance

Co1 may declare readiness when all boxes above can be ticked
with supporting evidence in repo and/or linked CoSuite repos.
